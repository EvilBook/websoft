S08

Reflection on results


This analysis was performed in a group. The participants Were Martin Zannato and Andrei Casian.

This is the LINK to the report on google docs.

We decided to analyze pages that we use daily in our lives and that are also very popular and compare them to each other to see how they performed. The pages we analyzed were imgur, github and youtube. Obviously these pages are extremely popular and there are some big companies behind them so they are very well optimized all over. The results we found were not that much surprising. Github performed the best overall because unlike youtube and imgur it is not that content heavy. The latter mentioned websites load images and videos on page load so they take at some points conciderably more resources than a more text oriente github.

Prposed optimizations for imgur:

Avoid an excessive DOM size. This would be hard to do because the page will have to sacrifice it's endless scrolling mode and continous image loading which makes for addictive behaviour but it would increase it's speed by a significant amount. Another advice would be to properly size images and that would save around 0.35s.

Proposed optimizations for github:

One suggestion is to serve static assets with an efficient cache policy. This is not a very quick fix since it requires some rework but it is very important and useful in the long term. It would greately increase page reload speeds.

Proposed optimizations for youtube:

A very inpactful suggestion for optimization is to limit the impact of third-party code on the page. Now youtube probably dont want to do this and are very well aware of it but limiting all the metric calculating applications and ads loading would help improve speeds in a big way.

One notably bad result was the sire imgur took almost 40s to fully load its main page which is a drastic difference from the other surveyed pages.


What is your TIL for this course section?


I learned about the existance of these very good tools for measuring web page performance and actually providing very good solutions on how to improve. This will be a priceless tool for our future web developement endeavours. I also learned that some big companies will sacrifice performance in order to achieve looks or some other functionality of their pages and that cannot be helped.
